前端架构与性能优化那些事儿

怎么样提高前端性能优化？(入门级)

  1. 小 将内容变小、请求后台次数变小等
  2. html、css、js内容压缩。
  3. 请求少了、文件大了，请求多了、文件小了。根据经验，请求不能超过5个，GZIP压缩大小在28.5k~32kb，撑死不能超过100kb。
  4. 一个域名并发多个cdn，减少cookie请求和响应
  5. 压缩请求报头

前端性能优化进阶

  1. 减少请求，缓存网络，本地走
  2. 离线缓存Mainifest（有问题，推新的东西推不出去）
  3. PWA
  4. AMP
  5. localStrorage可以存js文件，（最多存2.5M），是同步的
  6. Web SQL本地数据库，异步的，
  7. Service Workers一步到位的缓存，用于后期不再修改的项目

PC端性能优化

  1. lazyload :看不见的dom就不加载，
  2. 严格控制dom数量,一个div当几十个用(可以将html)
  3. 多多让GPU工作，减少CPU的工作量。
  4. 让GPU参与分层，可以参与分层的方式有：
    1. 使用transform
    2. 根元素
    3. postion
    4. 半透明
    5. 滤镜
    6. canvas
    7. video
    8. overflow
  5. 减小重排：
    1. 添加或删除元素
    2. 使用盒模型：box-sizing:border-box;
    3. 元素的内容
    4. 网页初始化时的dom渲染
    5. 如果元素有位置大小的变化就会引起重排，有颜色的变化就会有重绘

网页的渲染过程

  1. 获取当前网页的dom 分割成多层
  2. 对每个图层的节点结算样式结果
  3. 为每个节点形成图形和位置 回流 重排 layout
  4. 将每个节点绘制到图层的位图中 重绘 paint
  5. 图形

cpu和gpu的区别：

  1. 相同之处：两者都与外界有联系，都有自己的缓存体系，以及数字 和逻辑运算单元，一句话，它们都是为计算而生的，而不是为存储而生的。
  2. 不同之处：CPU主要负责操作系统和应用，GPU主要负责显示相关的数据。

重排和重绘：

重排：就是摆积木，

重绘：就是给积木图颜色

重排一定引起重绘，重绘不一定引起重排

注意点：

  1. javascript标签会阻塞dom渲染
  2. css不会阻塞dom渲染
  3. cdn的好处:
    1. 小
    2. 离你最近
    3. 能让前端突破限制
    4. 控制域名，预先请求，link标签的rel属性中加上dns-prefetch优化加载时间
    5. etag 对资源进行精准化的缓存

内存泄漏：没有用到的变量，和一直在使用着的变量就会造成内存泄漏

变量设为null并不能被及时的回收，想要及时的回收要用到WeakMap
let b = new Object();
let wm = new WeakMap();
wm.set(b,new Array(5*1024*1024));
b = null;
