架构师启蒙课第一节

单面和多页：

单页就是一个html带一堆js，spa

多页就是一堆html带一堆js,mpa

前端没有路由，前端的路由都是假的，现在的三大框架都是单页应用 。

代码流程:

  1. 创建项目目录

  2. 初始化项目目录

    npm init

  3. 安装webpack

    npm install webpack --save-dev
npm install webpack-cli --save-dev
-dev的意思是：测试环境，如果上线环境需要这个包，就去掉-dev
  4. 根目录下创建src目录，里面创建index.html和index.js文件。

  5. 在index.html中，写一个id为app的div标签

    <div id = "app"></div>

  6. 配置package.json文件，在scripts字段下添加配置代码

    "dev":"webpack --mode development",
"prod":"webpack --mode production"

  7. 在index.js中打印一句话：

    console.log("hello world");
  8. 终端执行npm run dev,可以看到webpack已经生成dist目录，并编辑好了main.js。
  9. 在index.html中引入main.js，并在浏览器打开，我们就会在控制台看见页面已经成功的执行了编辑后的js文件。如果我们执行npm run prod，就会编辑上线环境的代码，会看到main.js会被压缩成了一行了。

  10. 安装html-webpack-plugin

    npm i --save-dev html-webpack-plugin

  11. 配置webpack，根目录下创建webpack.config.js，并在里面添加上配置代码：

    const HtmlWebpackPlugin = require('html-webpack-plugin')

module.exports = {
  new HtmlWebpackPlugin({  // Also generate a test.html
       filename: 'index.html',//默认生成的路径，生成到dist里
       template: 'src/index.html'//页面路径
  }) 
｝

  12. 执行npm run dev 可以看到生成的dist目录里有两个文件，一个是Index.html一个是main.js。我们将index.html在浏览器中打开，发现已经成功执行。（注意，src下的html文件中，不需要引用index.js）

  13. 在src目录下新建components文件夹，在components目录下创建sync目录，并在sync目录下新建index.js(一定不要建成sync.js，会有意想不到的麻烦)和sync.css

  14. 在sync.js中写入代码:

    const sync = () => {
    console.log("this is sync");
}
export {
    sync
}

  15. 在src目录下的index.js这个主js文件中，引入暴露的sync接口。

    import {sync} from "./components/sync";//ES6解构
sync();
  16. 重新执行npm run dev后，刷新index.html，会看到控制台已经成功打印信息。
  17. webpack默认支持编辑js，如果是css和html就还需要重新配置。

  18. 安装支持css样式的包。

    npm install style-loader --save-dev
npm install css-loader --save-dev

  19. 配置webpack.config.js文件

    const HtmlWebpackPlugin = require('html-webpack-plugin')

module.exports = {
    module:{
        rules: [
            {
              test: /\.css$/,
              use: [ "style-loader", {
                loader: 'css-loader',
                options: {
                  modules: true,
                  localIdentName: '[path][name]__[local]--[hash:base64:5]'
                }
              } ]
            }
        ]
    },
    plugins: [
      new HtmlWebpackPlugin({  // Also generate a test.html
        filename: 'index.html',//默认生成的路径，生成到dist里
        template: 'src/index.html'//页面路径
    })  
  ]
}

  20. 在sync目录下的index.js文件中，导入main.css。（sync目录下的js文件，一定不能写成sync.js。会有意想不到的结果。）

    import item from "./sync.css";
document.getElementById("app").innerHTML=`<h1 class="${item.test}">大家好才是真的好</h1>`;
const sync = () => {
    console.log("sync");
}
export {
    sync
};
  21. 执行npm run dev。刷新页面，看到样式已经引入到了页面中。
  22. 实现前端工程化，需要区分开发环境和上线环境，项目根目录下创建config文件夹,并在里面创建开发路由webpack.development.js和上线路由webpack.production.js

  23. 安装判断开发环境还是上线环境yargs-parser和合并js的webpack-merge

    npm i yargs-parser --save-dev
npm install webpack-merge --save-dev

  24. webpack.config.js中使用yargs和关联config文件夹下的两个路由文件。导入merge，并对外暴露出去。

    const HtmlWebpackPlugin = require('html-webpack-plugin')
const argv = require('yargs-parser')(process.argv.slice(2))
//console.log("这是得到 的"+argv)
const _mode = argv.mode || "development";
const _mergeConfig = require(`./config/webpack.${_mode}.js`);
const merge = require('webpack-merge');
let webpackConfig = {
    module:{
        rules: [
            {
              test: /\.css$/,
              use: [ "style-loader", {
                loader: 'css-loader',
                options: {
                  modules: true,
                  localIdentName: '[path][name]__[local]--[hash:base64:5]'
                }
              } ]
            }
        ]
    },
    plugins: [
      new HtmlWebpackPlugin({  // Also generate a test.html
        filename: 'index.html',//默认生成的路径，生成到dist里
        template: 'src/index.html'//页面路径
    })  
  ]
}
module.exports = merge(_mergeConfig,webpackConfig);

  25. 运行npm run dev，正常执行，运行npm run prod也能正常执行。

  26. 将css从js中提取出来，安装mini-css-extract-plugin

    npm install mini-css-extract-plugin --save-dev

  27. 在webpack.config.js文件中引用mini

    const HtmlWebpackPlugin = require('html-webpack-plugin')
const argv = require('yargs-parser')(process.argv.slice(2))
//console.log("这是得到 的"+argv)
const _mode = argv.mode || "development";
const _mergeConfig = require(`./config/webpack.${_mode}.js`);
const merge = require('webpack-merge');
const MiniCssExtractPlugin = require("mini-css-extract-plugin");
let webpackConfig = {
    module:{
        rules: [
            {
              test: /\.css$/,
              use: [ {
                loader: MiniCssExtractPlugin.loader
              }, {
                loader: 'css-loader',
                options: {
                  modules: true,
                  localIdentName: '[path][name]__[local]--[hash:base64:5]'
                }
              } ]
            }
        ]
    },
    plugins: [
        new MiniCssExtractPlugin({
            filename: "[name].css",
            chunkFilename: "[id].css"
          }),
        new HtmlWebpackPlugin({  // Also generate a test.html
            filename: 'index.html',//默认生成的路径，生成到dist里
            template: 'src/index.html'//页面路径
    })  
  ]
}
module.exports = merge(_mergeConfig,webpackConfig);

  28. 我们发现在，所有的东西都生成在了dist文件夹里，如果时间长了，文件会很多，不方便管理，接下来要把文件归类，js文件生成到scripts包里，css文件生成到style包里。

    1. 修改mini配置代码：

      new MiniCssExtractPlugin({
    filename: "style/[name].css",//在文件名前加上style目录
    chunkFilename: "[id].css"
}),

    2. webpack.development.js中添加配置代码：

      module.exports = {
    output:{
        filename:"scripts/[name].bundles.js"
    }
};
  29. 删掉dist目录后重新执行npm run dev，会发现新生成的dist目录下多了两个文件夹：scripts和style，里面有对应的js和css文件。

  30. 配置上线环境，打hash：

    1. 修改mini配置代码：

      const HtmlWebpackPlugin = require('html-webpack-plugin')
const argv = require('yargs-parser')(process.argv.slice(2))
//console.log("这是得到 的"+argv)
const _mode = argv.mode || "development";
const _prodMode = (_mode == "production" ? true:false);
const _mergeConfig = require(`./config/webpack.${_mode}.js`);
const merge = require('webpack-merge');
const MiniCssExtractPlugin = require("mini-css-extract-plugin");
let webpackConfig = {
    module:{
        rules: [
            {
              test: /\.css$/,
              use: [ {
                loader: MiniCssExtractPlugin.loader
              }, {
                loader: 'css-loader',
                options: {
                  modules: true,
                  localIdentName: '[path][name]__[local]--[hash:base64:5]'
                }
              } ]
            }
        ]
    },
    plugins: [
        new MiniCssExtractPlugin({
            filename: _prodMode ? "styles/[name].[contenthash:5].css":"styles/[name].css",
            chunkFilename: _prodMode ? "styles/[id].[contenthash:5].css":"styles/[id].css"
        }),
        new HtmlWebpackPlugin({  // Also generate a test.html
            filename: 'index.html',//默认生成的路径，生成到dist里
            template: 'src/index.html'//页面路径
    })  
  ]
}
module.exports = merge(_mergeConfig,webpackConfig);

    2. webpack.production.js

      output: {
    filename: "scripts/[name].[contenthash:5].bundles.js"
},
  31. 这时，我们注意到，js压缩了，但是html和css并没有压缩，接下来我们就要压缩html和css。

  32. 在webpack.config.js文件中的plugins字段中添加html-webpack的配置，运行就会看到html文件已经被压缩

        new HtmlWebpackPlugin({  // Also generate a test.html
    filename: 'index.html',//默认生成的路径，生成到dist里
    template: 'src/index.html',//页面路径
    minify:{
        removeComments:_prodMode,
        collapseWhitespace:_prodMode
    }

    })

  33. 安装webpack-dev-server，它是用来启动一个后台服务器的，可以模拟后台接口数据，并进行时时监听，只要代码修改，就会刷新服务：

    npm install webpack-dev-server --save-dev

  34. 配置package.json，将dev-server配置到文件中。

    "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1",
    "dev": "webpack --mode development",
    "dev:server":"webpack-dev-server --mode development",
    "prod": "webpack --mode production"
},

  35. 配置webpack.confing.js

    devServer: {
    before(app) {
        app.get("/api/test", (req, res) => {
            res.json({
                code: 200,
                message: "Hello World"
            });
        });
    }
},
  36. 运行npm run dev:server就会启动一个端口服务，比如：localhost:8080,这时，我们就可以访问我们设置好的地址，也可以在后面加上我们写好的路由：/api/test，查看返回的json数据。

  37. 接下来我们就可以在sync下的index.js中请求这个接口，用来进行测试了，可以很方便的将后台给我们的接口数据按请求和解析的方式进行开发。

        import css from "./sync.css";
    document.getElementById("app").innerHTML=`<h1 class = "${css.test}">你妹的呀</h1>`;

       console.log("sync");

        .then(data=>{
            console.log("fethc结果:"+data.message);
            console.log("更新？");
        })

            console.log("🍎请稍事休息~");
            //发短信,收集错误
            //navigator.sendBeacon("http://www.aa.com/a.gif?errinfo="+err);
        });
    }
    export {
        sync
    }

  38. components目录下创建banner文件夹，里面创建index.js,并编写代码：

        const asyncbanner = {
        init(){
            console.log("banner");
        }
    }
    export default asyncbanner;

  39. 在src目录下的index.js主文件中导入这个js文件。

    import {sync} from "./components/sync/index";
//下面这行代码使用了魔法注释，如果不使用魔法注释，会发现在dist目录下生成的js文件名都是从0开始的，不方便识别。
import(/* webpackChunkName:"async-banner" */"./components/banner/index").then(_=>{
    _.default.init();
});
sync();

注意点：

  1. package.json文件中，有一些关键字不能乱动，如果乱改，会出现麻烦。这些关键字（例如test）可以使用npm test直接运行，其它自定义的需要使用npm run 自定义的名字 运行。 我们可以直接输入npm命令查看都有哪些关键字
  2. 现在向后台发请求，已经不再用ajax了，而是用fatch("要请求的地址")和navigator.sendBeacon("请求地址")。如果是做统计的时候，就用navigator.sendBeacon("")，它不占浏览器的并发。
